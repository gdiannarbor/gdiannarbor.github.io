What are the Options?

  1. Free DesktopServer: Local [on your computer] WP site creation prior to public launch on host server
  2. Hosting Control Panel: on yourLiveSite.com
  3. Manual WP Install: via FTP to your yourLiveSite.com
DesktopServer offers you the ability to work on your site locally on your computer before you publish it live to the world on the server that houses your hosted domain. To publish your site for others to see, you will need to upload it to a hosting server. You can use your hosting control panel to install WP or do this manually via FTP

Keeping your site secure is important. You'll need to purchase a quality SSL certificate if you're monetizing your site or if you're setting it up to attract social participation and networking with multiple users. The default use of admin as a uname and wp-admin as the name of your admin directory make your site more vulnerable to automated hacking. So we're going to change these now as a security best practice for protecting your WordPress website.

WordPress Security -> wp-login

Let's install our first Plugin

The default WP login page is wp-login.php

We're going to install a Plugin to alter this

In Dashboard -> Open Plugins

Click 'Add New' at the top

Enter 'rename wp-login' in the Search box -> Search Plugins

click 'Install Now'...[installing]...then 'Activate Plugin'

The Dashboard admin login code for all WordPress sites is in a file named wp-login.php - this is therefore a primary target for automated hacker scripts, so we're going to enable you to rename this login access path to add a layer of protection to your site. All of the available Security Plugins for WP are not created equal. I've experimented with some of them and most cause your site to load slower, which frustrates visitors. This Plugin is recommended by our friends at A2Hosting. It's light weight and effective so we're going to install it now.

WordPress Security -> Permalinks

Activation automagic opens Settings -> Permalinks

First change: from 'Default' to 'Post name'

Under Optional -> Rename wp-login.php

Enter something unique

'Save Changes'

Check out your new Dashboard url!

After activation you're going automagically going to land in Settings -> Permalinks. First we're going to change one of the standard Permalinks Settings from Default to Post name. This is important for two reasons - 1) this change makes your post and page url's [links] readable; and 2) it's necessary to enable the next Permalink change we're going to make to add security to your site by changing the location name of your Dashboard login url [link]. You can pick any word or phrase as the replacement name for your login address. The default entry from wp-login to 'login' is just a prompt that tells you you're changing the address of your login URL / link.

What's the Dif??

Let's start with Pages

Pages: content = quick and easy access for visitors

'Static' page - term refers to Navigation

Pages: can contain interactive content and media

Pages are for content that you want your visitors to always find quick and easy. The industry term for this 'always available' content via Pages is 'static'. However static refers to the navigation - you can include interactive dynamic content and media in your Pages.

Users

What are User Roles?

Users editing privileges.

  • Administrator: access to all administration features
  • Editor: publish and manage posts including posts from other users
  • Author: publish and manage own posts
  • Contributor: write and manage own posts, publish requires approval
  • Subscriber: can only manage their profile

The Power of Plugins

Plugins add functionality to WordPress
Examples:
ecommerce, SEO, forms, photo galleries, social media sharing, forums and much more!

WordPress includes 2 plugins:

  • Akismet: Protects your site from comment and trackback spam. *Note: requires an Akismet key and monthly subscription fee.
